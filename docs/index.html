The problem

Running a lot of AI is running blind

The more agents you run in parallel, the less you can see. Two things always go wrong.

⌨

A prompt gets buried

One agent quietly stops and waits for your approval while you're heads-down in another pane. You find out ten minutes later — ten minutes it did nothing.

💸

FAQ

Questions, answered

Does my code or prompt data leave my machine?

No. The open-source core runs entirely locally — collectors, dashboard, MCP server, alerts. Nothing is sent anywhere. Paid tiers add an optional hosted relay for push/history, and even then only session metadata (status, token counts, cost) travels — never your code or prompts.

Which tools does it detect?

Claude Code natively (hooks + transcripts) for exact status, mode, and tokens. Other AI CLIs — aider, codex, gemini-cli, cursor-agent, goose, opencode — via process scan. And anything you write, via the heartbeat SDK.

How does the "waiting for input" alert work?

For Claude Code it's exact: its Notification hook fires the instant it needs you, and AgentDeck flashes the card, rings a bell, shows a desktop toast, and (if configured) pushes to your phone via ntfy. For other tools it's inferred from process and heartbeat signals.

Is the cost number real money?

By default it's a live estimate from token counts against a pricing table you can override. Turn on the usage collector with a provider admin key and it also shows your real billed spend for the day.

What do I need to run it?

Node 20+. It's built for WSL and works on Linux and macOS. The web dashboard opens in any browser; the TUI runs in any terminal.
